  	实现一个简易21点游戏

1. 游戏需要一个玩家，一个庄家。

2. 游戏需要一副扑克牌（共52张，不包括大王，小王）。

3. 对于数字牌[2-10]，数字牌代表的点数与其牌本身点数相同，如黑桃2代表2点。

4. 对于非数字牌{A, J, Q, K}, A代表的点数为1，J, K和Q代表的点数为10，如黑桃A为1点，黑桃J为10点。

6. 游戏开始时，庄家和玩家随机获得两张牌，牌面朝下。

7. 游戏开始后，首先选定玩家为发话者

8. 发话者发话，可以有两种选择，即拿牌(Hit)和停牌(Stand)。如选择拿牌，则到9，停牌则到10

9. 发话者随机得到一张牌，牌面朝上，并统计发话者手中的牌的所有点数和，可能为如下三种情况:
  (1) 小于21点(<21)，则到8；
  (2) 等于21点(==21)，发话者胜，游戏结束；
  (3) 大于21点(>21)，发话者爆点，非发话者胜，游戏结束;

10. 如当前发话者为玩家，则改变发话者为庄家，到8。如当前发话者为庄家，则统计双方点数，点数高者为胜，点数相等为平局，游戏结束。

要求应用到如下语言，技术，工具或实践（其中HTML, jQuery, CSS等需同学们自学，大家可以参考Baseline里面的学习资料）：

1. 用Java语言进行开发，使用Gradle构建自己的项目。

2. 可以采用Guava进行必要的处理。

3. 需要有Web界面，允许使用JSP, HTML, JQuery, CSS等页面相关技术。

4. 要求采用Spring MVC。

5. 要求代码有测试，并且按照TDD进行开发，单元测试可能会mock，使用时要求使用mockito。

6. 使用Git做版本管理（大家可以传到github上，和buddy一起review代码）。

7. 要求程序有良好的异常处理机制。
